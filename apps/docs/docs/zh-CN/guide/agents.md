# 智能体

`DocPlaybook` 构建在 `LLM` 智能体之上，而不仅仅是一次性的翻译调用。

智能体负责翻译文档、从评审中学习、维护可重用的指导文件，并根据这些指导对翻译输出进行 `Lint` 检查。

## 智能体维护的内容

`DocPlaybook` 保持两层由 AI 维护的指导：

- `.docplaybook/playbook.md`
- `.docplaybook/memories/<target>.md`

智能体会随着时间更新这些文件，以便未来的翻译和 lint 运行可以重用之前的决策。

## 智能体职责

### 翻译

翻译智能体：

- 读取源文档
- 读取全局 `playbook` 和目标语言记忆
- 检查每个目标文档的源哈希是否更改
- 跳过已是最新的目标
- 当源发生变化或目标缺失时，刷新整个目标文档

### 学习

学习智能体：

- 检查自上次学习运行以来目标哈希是否更改
- 跳过在当前版本已学习过的目标
- 让 LLM 从当前的源/目标对中提取可重用的指导
- 更新 `.docplaybook/playbook.md`，加入语言无关的规则
- 更新 `.docplaybook/memories/<target>.md`，加入语言特定的术语和样式说明

### Lint

Lint 智能体：

- 读取源文档
- 读取已翻译文档
- 读取全局 `playbook` 和目标语言记忆
- 在多个维度上对质量进行评分
- 输出明确的发现
- 可以使用 `--fix` 应用安全修复

## 全局与语言特定的指导

智能体有意保持这种划分：

- `playbook.md`
  - 语言无关的指导
  - 统一的语调
  - 受保护词汇
  - 全项目范围的翻译规则
- `memories/<target>.md`
  - 语言特定的术语
  - 语言特定的样式说明

这样可以保持首个版本的简洁，同时允许智能体随着时间学习有用的规则。

## 选择此模型的原因

基于智能体的结构有助于 `DocPlaybook` 在多次运行中保持有用：

- 翻译变得更加一致
- 人工审核的投入会逐渐积累，而不是流失
- `Lint` 有更清晰的评判标准
- 指导内容以 Markdown 存储，因而可编辑
