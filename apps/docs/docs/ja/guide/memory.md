# メモリ

DocPlaybookは翻訳ガイダンスをMarkdownとして保持します。これによりエージェントが再利用でき、人間が編集可能な状態を保てます。

## ファイル

DocPlaybookは2つのレベルのメモリを管理します:

- `.docplaybook/playbook.md`
- `.docplaybook/memories/<target>.md`

最初はグローバルで、2つ目は各ターゲット言語ごとに特化しています。

## グローバルプレイブック

`.docplaybook/playbook.md` は言語非依存のガイダンス用です。

以下を保持します:

- `## Voice`
- `## Protected Terms`
- `## Translation Rules`

典型的な例:

- ドキュメントは技術的で直接的な調子を保つべきです
- 製品名とCLIコマンドは翻訳すべきではありません
- 警告は和らげないでください
- Markdownの構造はそのままにしてください

## 言語別メモリ

各 `.docplaybook/memories/<target>.md` ファイルは言語固有のガイダンス用です。

以下を保持します:

- `## Terminology`
- `## Style Notes`

典型的な例：

- 繰り返し出てくる技術用語の推奨翻訳
- 特定の英語表現を翻訳せずそのまま残すかどうか
- 対象言語に応じたトーンや丁寧さの選択
- 句読点や表現の好み（その対象言語に合わせたもの）

## メモリの利用方法

翻訳とリンティングの過程で、DocPlaybook は次を組み合わせます：

1. グローバルな playbook
2. 対象言語のメモリ

その組み合わせたコンテキストが、現在の実行で再利用可能な翻訳基準となります。

## メモリの更新方法

`docplaybook learn` はメモリが更新される主な方法です。

人間が翻訳済みのドキュメントを編集すると、学習フローは次のようになります：

1. 現在のターゲットファイルのハッシュが既に学習済みかどうかを確認します
2. 現在のソースドキュメントとターゲットドキュメントを読み取ります
3. LLM に、どの所見が再利用可能なガイダンスで、どれが無視すべきかを尋ねます
4. 言語に依存しない教訓を `playbook.md` に更新します
5. 言語固有の用語とスタイルに関するノートで `memories/<target>.md` を更新します

これによりレビューの労力が時間とともに蓄積されます。

## ブートストラップがメモリを初期化する方法

既に翻訳済みのページがある既存のドキュメントサイトでは、`docplaybook bootstrap --langs ...` がリポジトリ内のソース/ターゲットで整列した例から最初のメモリファイルを作成します。

プロジェクトに既に翻訳されたドキュメントが含まれている場合、`init` の後の推奨される最初のステップです。

## 構造が最小限である理由

最初のバージョンでは、メモリを意図的に小規模に保ちます：

- グローバルルールは `playbook.md` に配置します
- 言語固有のルールは `memories/<target>.md` に配置します

これによりエージェントは次のことを行いやすくなります:

- ファイルを読む
- 安全に更新する
- 簡潔に保つ
- ルールをあまり多くのセクションに重複させない
