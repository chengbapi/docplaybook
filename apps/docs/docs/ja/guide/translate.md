# 翻訳

`translate` はターゲットドキュメントをソースドキュメントと整合させます。

## 機能

各ターゲットドキュメントに対して、DocPlaybook:

1. 現在のソースドキュメントを解析する
2. 現在のソースハッシュを計算する
3. そのハッシュをターゲットの最終保存ハッシュと比較する
4. ハッシュが変わらずターゲットが既に存在する場合はスキップする
5. ソースが変更されたかターゲットが存在しない場合はターゲット記事を更新する

これは構造を意識した書き戻しを行う記事レベルの翻訳です。Git の差分ベースの翻訳ではありません。

## 典型的な使用法

構成済みのすべてのターゲット言語を最新化する:

```bash
pnpm exec docplaybook translate .
```

処理を1つのロケールに限定する:

```bash
pnpm exec docplaybook translate . --langs ja
```

完全なリフレッシュを強制する:

```bash
pnpm exec docplaybook translate . --force
```

または明示的なエイリアスを使う:

```bash
pnpm exec docplaybook retranslate .
```

## 安全に保持されるもの

処理の判定はドキュメントレベルで行われますが、翻訳はブロックを意識したままです。

DocPlaybook は次を保持します:

- Markdown の構造
- コードフェンス
- フロントマターの構造
- 保護されたフレームワーク固有のフィールド

`Rspress` の場合、これは `layout.kind = "rspress"` で扱われるフレームワーク固有のリソースも含みます:

- `docs/<lang>/...` 配下の通常ドキュメント
- `_nav.json`
- `_meta.json`
- `i18n.json` のロケールエントリ
- `index.md` のホワイトリスト化されたホームページのフロントマター項目
