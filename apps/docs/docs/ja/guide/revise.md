# 改訂

`learn`はレビュワーの編集を再利用可能なガイダンスに変換します。

## 何をするか

各ターゲットドキュメントについて、`DocPlaybook`は次のことを行います：

1. 現在のターゲットのハッシュが既に学習されているかを確認します
2. 同じバージョンが既に処理されている場合はそのターゲットをスキップします
3. 現在のソースとターゲットのドキュメントを読み取ります
4. どの観察結果を再利用可能なガイダンスとするかをモデルに尋ねます
5. 更新します:
   - `.docplaybook/playbook.md`
   - `.docplaybook/memories/<lang>.md`

これは`learn`が現在のソース/ターゲットのペアに基づいており、Gitの前後差分には基づかないことを意味します。

## いつ実行するか

`learn`はレビュワーが翻訳されたドキュメントに重要な編集を行った後に使用します。

典型的なケース:

- 用語の修正をプロジェクト全体のガイダンスにしたい場合
- ターゲット言語のスタイルに関する決定を再利用したい場合
- レビュワーが特定のロケールでの製品用語の表記を明確にした場合

## 典型的な使用法

設定されたすべてのターゲット言語で学習する：

```bash
pnpm exec docplaybook learn .
```

1つのロケールのみを学習する：

```bash
pnpm exec docplaybook learn . --langs ja
```

再学習を強制する：

```bash
pnpm exec docplaybook learn . --force
```

または明示的なエイリアスを使って：

```bash
pnpm exec docplaybook relearn .
```

## ガイダンスの分割

この区別を明確に保ってください：

- `playbook.md`はプロジェクト共通の知識を格納します
- `memories/<lang>.md`はターゲット言語向けのガイダンスを格納します
- `.docplaybook/state/*.json`はブランチの進捗のみを保存します
