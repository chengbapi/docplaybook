# エージェント

DocPlaybook は一回限りの翻訳呼び出しではなく、LLM エージェントを中心に構築されています。

エージェントはドキュメントの翻訳、レビューからの学習、再利用可能なガイダンスファイルの管理、そしてそのガイダンスに対する翻訳出力のリンティングを担当します。

## エージェントが管理するもの

DocPlaybook は AI が管理するガイダンスを二層で保持します：

- `.docplaybook/playbook.md`
- `.docplaybook/memories/<target>.md`

エージェントはこれらのファイルを時間をかけて更新し、将来の翻訳やリンティングの実行が以前の判断を再利用できるようにします。

## エージェントの責務

### 翻訳

翻訳エージェントは次のことを行います：

- ソースドキュメントを読み取る
- グローバルプレイブックとターゲット言語のメモリを読み取る
- 各ターゲットドキュメントのソースハッシュが変更されたか確認する
- 既に最新のターゲットはスキップする
- ソースが変更されたかターゲットが存在しない場合、ターゲットドキュメント全体を更新する

### 学習

学習エージェントは次のことを行います：

- 前回の学習実行以降でターゲットのハッシュが変更されたか確認する
- 現在のバージョンですでに学習済みのターゲットはスキップする
- 現在のソース/ターゲットのペアから LLM に再利用可能なガイダンスを抽出させる
- 言語に依存しないルールで `.docplaybook/playbook.md` を更新する
- 言語固有の用語およびスタイルノートで `.docplaybook/memories/<target>.md` を更新する

### リント

リントエージェントは次のことを行います：

- ソースドキュメントを読み取る
- 翻訳済みドキュメントを読み取る
- グローバルプレイブックとターゲット言語のメモリを読み取る
- 複数の次元で品質を評価する
- 明確な所見を出力する
- `--fix` で安全な修正を適用できる

## グローバルと言語固有のガイダンス

エージェントはこの分離を意図的に維持します：

- `playbook.md`
  - 言語に依存しないガイダンス
  - 共通のトーン
  - 保護された用語
  - プロジェクト全体の翻訳ルール
- `memories/<target>.md`
  - 言語固有の用語
  - 言語固有のスタイルノート

これにより最初のバージョンはシンプルに保たれつつ、エージェントが時間をかけて有用なルールを学習できるようになります。

## このモデルを採用する理由

このエージェントベースの構成は、DocPlaybook が繰り返しの実行でも有用であり続けるのに役立ちます：

- 翻訳の一貫性が向上する
- 人間によるレビューの労力が蓄積され、失われずに活用される
- リントが判断するための基準が明確になる
- ガイダンスは Markdown に保存されるため、編集可能なままである
