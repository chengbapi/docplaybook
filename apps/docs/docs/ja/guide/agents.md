# エージェント

DocPlaybookは一回限りの翻訳呼び出しではなく、LLMエージェントを中心に構築されています。

エージェントはドキュメントの翻訳、レビューからの学習、再利用可能なガイダンスファイルの管理、翻訳結果がそのガイダンスに従っているかのリンティングを担当します。

## エージェントが管理するもの

DocPlaybookはAIが管理するガイダンスを2層で保持します：

- `.docplaybook/playbook.md`
- `.docplaybook/memories/<target>.md`

エージェントはこれらのファイルを時間をかけて更新し、将来の翻訳やリンティング実行で以前の判断を再利用できるようにします。

## エージェントの責務

### 翻訳

翻訳エージェントは次のことを行います：

- ソースドキュメントを読み取る
- グローバルな playbook とターゲット言語のメモリを読み取る
- 再生成が必要な箇所のみを翻訳する
- 残りの翻訳済みドキュメントはそのまま保持する

### 学習

学習エージェントは次のことを行います：

- Git の `HEAD` にある翻訳済みファイルと現在の作業ツリーのバージョンを比較する
- 構造がまだ整っている場合に、変更された翻訳可能ブロックを抽出する
- それらの編集が再利用可能な修正かどうかをLLMに判断させる
- 言語非依存のルールを `.docplaybook/playbook.md` に更新する
- 言語固有の用語とスタイルノートを `.docplaybook/memories/<target>.md` に更新する

### リント

リントエージェントは次のことを行います：

- ソースドキュメントを読み取る
- 翻訳済みドキュメントを読み取る
- グローバルな playbook とターゲット言語のメモリを読み取る
- 複数の観点で品質を評価する
- 明確な所見を出力する
- `--fix` オプションで安全な修正を適用できる

## グローバルなガイダンスと言語固有のガイダンス

エージェントはこの分離を意図的に維持しています：

- `playbook.md`
  - 言語非依存のガイダンス
  - 共通のボイス
  - 保護された用語
  - プロジェクト全体の翻訳ルール
- `memories/<target>.md`
  - 言語固有の用語
  - 言語固有のスタイルノート

これにより最初のバージョンはシンプルなままにしつつ、エージェントが時間をかけて有用なルールを学習できるようにします。

## このモデルを採用する理由

このエージェントベースの構造により、DocPlaybookは繰り返し実行しても有用性を維持します：

- 翻訳の一貫性が高まる
- 人間によるレビューの労力が蓄積され、失われない
- リントが評価するための基準が明確になる
- ガイダンスはMarkdownで保存されるため編集可能なままである
