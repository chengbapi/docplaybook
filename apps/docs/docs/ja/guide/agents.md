# エージェント

DocPlaybookは一回限りの翻訳呼び出しではなく、LLMエージェントを中心に設計されています。

エージェントは、ドキュメントの翻訳、レビューからの学習、再利用可能なガイダンスファイルの維持、そして翻訳結果がそのガイダンスに準拠しているかのリンティングを担います。

## エージェントが管理するもの

DocPlaybookはAIが維持するガイダンスを2層で管理します：

- `.docplaybook/playbook.md`
- `.docplaybook/memories/<target>.md`

エージェントはこれらのファイルを時間をかけて更新し、以後の翻訳やlintの実行が過去の判断を再利用できるようにします。

## エージェントの責務

### 翻訳

翻訳エージェントは次のことを行います：

- ソース文書を読み取る
- グローバルなplaybookとターゲット言語のメモリを読み取る
- 各ターゲット文書についてソースのハッシュが変更されたか確認する
- 既に最新のターゲットはスキップする
- ソースが変更されたかターゲットが存在しない場合、ターゲット文書全体を更新する

### 学習

学習エージェントは次を行います：

- 前回の学習実行以降にターゲットのハッシュが変わったか確認する
- 現行バージョンですでに学習済みのターゲットはスキップする
- 現在のソース／ターゲットのペアから再利用可能なガイダンスをLLMに抽出させる
- 言語非依存のルールで `.docplaybook/playbook.md` を更新する
- 言語固有の用語やスタイル注記で `.docplaybook/memories/<target>.md` を更新する

### リント

リントエージェントは次を行います：

- ソース文書を読み取る
- 翻訳済み文書を読み取る
- グローバルなplaybookとターゲット言語のメモリを読み取る
- 複数の観点で品質を評価する
- 明示的な所見を出力する
- `--fix` で安全な修正を適用できる

## グローバルと言語固有のガイダンス

エージェントはこの分離を意図的に維持します：

- `playbook.md`
  - 言語非依存のガイダンス
  - 共有の文体
  - 保護された用語
  - プロジェクト全体の翻訳ルール
- `memories/<target>.md`
  - 言語固有の用語
  - 言語固有のスタイル注記

これにより最初のバージョンはシンプルに保たれつつ、エージェントが時間とともに有用なルールを学習できるようになります。

## このモデルを採用する理由

このエージェントベースの構成は、DocPlaybookが繰り返しの実行でも有用であり続けるのに役立ちます：

- 翻訳がより一貫性を持つ
- 人的レビューの労力が蓄積され、失われにくくなる
- リンターが判断するための基準が明確になる
- ガイダンスはMarkdownで保存されるため編集可能なまま維持される
